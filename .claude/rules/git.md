# Git ワークフロー

## ブランチ

- 作業ブランチは `claude/` プレフィックスで作成されているか？
- `main` ブランチに直接 push していないか？

## コミットメッセージ

- プレフィックスは `feat:` / `fix:` / `docs:` / `refactor:` のいずれかか？
- 1コミット＝1つの変更目的か？（複数の独立した変更を混在させていないか）
- コミットメッセージの末尾にセッションURLを含めているか？

## Push

- `git push -u origin <branch-name>` の形式で push しているか？
- ネットワークエラー時のみリトライし、最大4回（2s / 4s / 8s / 16s）で止めているか？
- ユーザーに明示的に依頼されるまで PR を作成していないか？

## 破壊的操作

- `--force` push・`reset --hard`・`checkout .` を実行する前にユーザーに確認したか？
- `--no-verify` でフックをスキップしていないか？
