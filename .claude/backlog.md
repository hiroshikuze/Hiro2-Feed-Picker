## Bot送信リンクのクリック分析と記事選定への自動フィードバック

### 概要

LINEでBotが送信したリンクのクリック数を計測し、分析結果を次回以降の記事抽出・要約の優先度に自動反映する。

### 想定フロー

```
リンククリック計測
  └── 短縮URL（例: Google Cloud Run Functions + Firestore）でクリックをインターセプト
        ↓
Googleスプレッドシートへ書き込み（リンクURL・クリック日時・ユーザーID）
        ↓
定期バッチで集計（クリック率の高いドメイン・キーワードを抽出）
        ↓
GeminiプロンプトまたはキーワードシートへFeedback
  ├── クリック率の高いキーワード → 優先キーワードとして重み付け
  └── クリック率の低いソース → フィルタリング候補として提示
```

### 検討事項

- **クリック計測方法**: LINE Messaging APIはリンクのクリックイベントを直接取得できないため、リダイレクト方式の中継URLが必要
  - 選択肢A: Google Cloud Run FunctionsでリダイレクターURLを実装
  - 選択肢B: bit.lyなどの外部短縮URLサービスのAPI活用（プライバシー・コスト要確認）
- **フィードバックの粒度**: キーワード単位か、RSSソース（ドメイン）単位か、両方か
- **自動化 vs 確認フロー**: 完全自動でキーワードシートを書き換えるか、オーナーにLINEで提案して承認後に反映するか
- **プライバシー**: ユーザーIDとクリック履歴の紐付けについてプライバシーポリシーの整備が必要

### 関連アーキテクチャへの影響

- `appsscript.json` のOAuthスコープ追加が必要になる可能性あり
- 新たなスプレッドシートシート（例: `click_log`）の追加
- GAS外部サービス（Cloud Run等）との連携が発生する場合、`main.js` の責務分離を検討
